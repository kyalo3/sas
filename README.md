## ğŸŒŸ Overview

Sustain a Share is a comprehensive, full-stack food donation management platform that connects donors, recipients, volunteers, and administrators in an efficient ecosystem for managing food donations. The platform facilitates the entire donation lifecycle from request creation to fulfillment, ensuring transparency and accountability throughout the process.

Built with modern web technologies, the platform leverages FastAPI's high-performance asynchronous capabilities on the backend and React's component-based architecture on the frontend to deliver a seamless, responsive user experience.

## ğŸ¯ Project Purpose

This platform addresses food insecurity by:
- Enabling recipients to request specific food items
- Allowing donors (individuals and organizations) to fund approved requests
- Providing volunteers with visibility into all donation activities
- Giving administrators control over donation approval and system management
- Creating a transparent, trackable system for food distribution

### Architecture Components

#### **Frontend Layer (React Application)**
- **Purpose**: User interface and interaction
- **Port**: 3000 (Development)
- **Key Features**:
  - Single Page Application (SPA) architecture
  - Client-side routing with React Router
  - Component-based UI with reusable elements
  - State management using React Hooks
  - Responsive design for all screen sizes
  - JWT token storage in localStorage
  - Axios HTTP client for API communication

#### **Backend Layer (FastAPI Application)**
- **Purpose**: Business logic and API endpoints
- **Port**: 8000
- **Key Features**:
  - Asynchronous request handling
  - RESTful API design
  - JWT-based authentication
  - Role-based authorization
  - Automatic API documentation (Swagger/ReDoc)
  - Input validation with Pydantic
  - Error handling and logging
  - CORS middleware for cross-origin requests

#### **Database Layer (MongoDB Atlas)**
- **Purpose**: Data persistence and retrieval
- **Type**: NoSQL Document Database (Cloud-hosted)
- **Key Features**:
  - Flexible schema design
  - Async operations with Motor driver
  - Automatic replication and backups
  - Built-in security and encryption
  - Scalable cloud infrastructure
  - Real-time data synchronization

### User Roles & Capabilities

#### 1. **Donors**
- Create and manage donor profiles (company details, services offered, locations)
- Browse approved donation requests from recipients
- Fund donation requests
- Track funded donations
- View donation history

#### 2. **Recipients**
- Create recipient profiles (household information, needs assessment)
- Submit donation requests (food items, quantities, descriptions)
- View request status (pending, approved, funded, completed, rejected)
- Track received donations

#### 3. **Volunteers**
- View all donations in the system
- Monitor donation statuses
- Coordinate between donors and recipients
- Access comprehensive donation data

#### 4. **Administrators**
- Approve or reject donation requests
- Manage all users in the system
- Monitor platform activity
- Oversee donation lifecycle

### Core Functionality

- **Authentication & Authorization**: Secure JWT-based authentication with role-based access control
- **Donation Management**: Complete CRUD operations for donations with status tracking
- **Profile Management**: Separate detailed profiles for donors, recipients, and volunteers
- **Review System**: Allow users to leave feedback and reviews
- **Real-time Updates**: Dynamic dashboards that reflect current donation states

##### 2.4 Configure MongoDB Atlas

1. **Create MongoDB Atlas Account**
   - Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas/register)
   - Sign up for a free account

2. **Create a Cluster**
   - Choose "Build a Database"
   - Select "Shared" (Free tier)
   - Choose your cloud provider and region
   - Click "Create Cluster"

3. **Create Database User**
   - Go to "Database Access"
   - Click "Add New Database User"
   - Choose authentication method: Username/Password
   - Create username and password (save these!)
   - Set user privileges to "Read and write to any database"

4. **Configure Network Access**
   - Go to "Network Access"
   - Click "Add IP Address"
   - For development: Click "Allow Access from Anywhere" (0.0.0.0/0)
   - For production: Add your specific IP addresses

5. **Get Connection String**
   - Go to "Database" â†’ "Connect"
   - Choose "Connect your application"
   - Copy the connection string
   - Format: `mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/<database>?retryWrites=true&w=majority`

6. **Update Database Configuration**

Edit `app/database.py`:

```python
from motor.motor_asyncio import AsyncIOMotorClient

# MongoDB connection details
MONGO_DETAILS = "mongodb+srv://your_username:your_password@cluster0.xxxxx.mongodb.net/FoodDonation?retryWrites=true&w=majority"

# Establish connection to MongoDB server
client = AsyncIOMotorClient(MONGO_DETAILS)

# Initialize database
database = client.FoodDonation

# Initialize collections
donor_collection = database.get_collection("donors")
recipient_collection = database.get_collection("recipients")
donation_collection = database.get_collection("donations")
user_collection = database.get_collection("users")
volunteer_collection = database.get_collection("volunteers")
review_collection = database.get_collection("reviews")
```

**Replace:**
- `your_username` with your MongoDB username
- `your_password` with your MongoDB password
- `cluster0.xxxxx` with your actual cluster address
- `FoodDonation` with your preferred database name

### Troubleshooting Common Issues

#### Backend Issues

**1. Port 8000 already in use:**
```bash
# Find process using port 8000
lsof -ti:8000 | xargs kill -9  # Linux/macOS
netstat -ano | findstr :8000   # Windows

# Or use a different port
uvicorn app.main:app --reload --port 8001
```

**2. MongoDB connection timeout:**
- Check your internet connection
- Verify MongoDB Atlas IP whitelist includes your IP
- Confirm username and password are correct
- Check connection string format

**3. Import errors:**
```bash
# Reinstall dependencies
pip install -r requirements.txt --force-reinstall
```

#### Frontend Issues

**1. Port 3000 already in use:**
```bash
# Kill process on port 3000
lsof -ti:3000 | xargs kill -9  # Linux/macOS

# Or use different port
PORT=3001 npm start
```

**2. Node modules errors:**
```bash
# Delete and reinstall
rm -rf node_modules package-lock.json
npm install
```

**3. CORS errors:**
- Ensure backend is running
- Check CORS configuration in `app/main.py`
- Verify API URL in frontend components

- CORS: Restrict to specific domain
- Environment variables in secure location
